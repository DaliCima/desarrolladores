complemento de estados de cuenta de combustibles.

 

Complemento que incluye la información aplicable al estado de cuenta emitido por un prestador de servicios de monedero electrónico.

Puedes descargar la documentación y definición del complemento en el sitio del SAT.

Datos del complemento de estados de cuenta de combustibles

A continuación una descripción y ejemplo de cada atributo en el complemento. La declaración de nameSpace es a nivel primario en nodo Comprobante. xmlns:ecc="http://www.sat.gob.mx/ecc"

  • EstadoDeCuentaCombustible

    • tipoOperacion : Tarjeta
      Atributo requerido para expresar el tipo de operación de acuerdo con el medio de pago, valor fijado en "Tarjeta".
    • numeroDeCuenta : 78738912
      Requerido para expresar el número de cuenta del adquirente del monedero electrónico.
    • subTotal : 1200.00
      Atributo opcional para presentar la suma de todos los importes tipo ConceptoEstadoDeCuentaCombustible.
    • total : 1392
      Atributo requerido para expresar el monto total de consumo de combustible.

  • Conceptos

    • ConceptoEstadoDeCuentaCombustible
    • identificador : 893827162
      Atributo requerido para la expresión del identificador o número del monedero electrónico.
    • fecha : 2014-07-28T14:30:00
      Atributo requerido para la expresión de la Fecha y hora de expedición de la operación reportada. Se expresa en la forma aaaa-mm-ddThh:mm:ss, de acuerdo con la especificación ISO 8601.
    • rfc : AAA010101AAA
      Atributo requerido del RFC del enajenante del combustible.
    • claveEstacion : 6732
      Atributo requerido para expresar la clave de cliente de la estación de servicio, a 10 caracteres, cuando sea requerido.
    • cantidad : 120.00
      Atributo requerido para definir el volumen de combustible adquirido.
    • nombreCombustible : Magna
      Atributo requerido para expresar el nombre del combustible adquirido.
    • folioOperacion : 82191
      Atributo requerido para referir el número de folio de cada operación realizada por cada monedero electrónico.
    • valorUnitario : 10.00
      Atributo requerido para definir el precio unitario del combustible adquirido.
    • importe : 1200.00
      Atributo requerido para definir el monto total de consumo de combustible. Debe ser equivalente al resultado de multiplicar la cantidad por el valor unitario.
    • Traslados
      • Traslado
      • impuesto : IVA
        Atributo requerido para definir el tipo de impuesto trasladado, valores aceptados "IVA", "IEPS".
      • tasa : 16.00
        Atributo requerido para señalar la tasa del impuesto que se traslada por cada concepto amparado en el comprobante.
      • importe : 192.00
        Atributo requerido para definir el importe o monto del impuesto trasladado.

* Los campos en negritas indican atributos obligatorios

ejemplo

<cfdi:Complemento>
  <ecc:EstadoDeCuentaCombustible tipoOperacion="Tarjeta" numeroDeCuenta="78738912" total="0.00">
      <ecc:Conceptos>
          <ecc:ConceptoEstadoDeCuentaCombustible identificador="893827162"
              fecha="2014-07-28T14:30:00" rfc="AAA010101AAA" claveEstacion="6732"
              cantidad="120.00" nombreCombustible="Magna" folioOperacion="82191"
              valorUnitario="10.00" importe="1200.00">
              <ecc:Traslados>
                  <ecc:Traslado impuesto="IVA" tasa="0.00" importe="0.00"/>
                  <ecc:Traslado impuesto="IVA" tasa="0.00" importe="0.00"/>
              </ecc:Traslados>
          </ecc:ConceptoEstadoDeCuentaCombustible>
      </ecc:Conceptos>
  </ecc:EstadoDeCuentaCombustible>
</cfdi:Complemento>
